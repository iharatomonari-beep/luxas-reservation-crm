# LUXAS 予約・顧客管理システム — Claude Code 引き継ぎ手順

## 最優先の常設ルール（毎回・必ず最初に）

Claude Code はこのプロジェクトで作業を始めるとき、**他の何よりも先に次の順で読む**。

1. `docs/instructions/_execution-policy.md` … 実行ポリシー（常設ルール）。**これを読まずに `_index.md` や個別タスクを実行してはいけない。**
2. `docs/instructions/_index.md` … 現在の実行キュー。状態が「未実行」のタスクを上から（最小番号から）1つずつ処理する。

実行は各タスクの**安全レベル**に従う（詳細は `_execution-policy.md`）。

- **SAFE** … 確認なしで自動実行してよい。完了後に報告。
- **CAUTION** … 実行前に停止して確認を求める。ユーザーの許可が出るまで着手しない。
- **STOP** … 実行せず停止して報告する。

共通の停止条件: エラー（lint / build / typecheck 失敗）、仕様判断が必要、実機確認が必要、依存タスク未完了、指示にないファイル変更が必要。これらに当たったら止めて報告し、勝手に次へ進まない。すべてのタスクで確認を求める必要はない（危ない作業＝CAUTION/STOP だけ止める）。

ユーザーからの短い指示「`_execution-policy.md` と `_index.md` を読んで実行して」は、上記の運用を指す。

## セッション開始時に必ずやること

1. `docs/sessions/` フォルダの最新ファイルを読む（日付降順で最新1〜2件）
2. `docs/instructions/_index.md` を読み、未実行の指示があるか確認する
3. 前回の「次にやること」を確認する
4. 現在の実装状態をユーザーに要約して伝えてから作業を始める

```
docs/sessions/YYYY-MM-DD.md      ← 最新ファイルから読む
docs/instructions/_index.md      ← Cowork からの未実行指示を確認
```

## Cowork からの実装指示の扱い（キュー方式・順番に全部こなす）

- 実装指示は Cowork（設計担当）が `docs/instructions/` に貯めていく。複数たまっていることが前提。
- ユーザーが「未実行の指示を上から順に全部実行して」と言ったら、`docs/instructions/_index.md` の **status が「未実行」の指示を番号順に1つずつ** こなす。
- **1つ実行 → 完了条件（自己検証）を全て満たす → 結果を報告 → `_index.md` の status を「完了」に更新 → `docs/sessions/` に追記 → 次の未実行へ進む**、を未実行が無くなるまで繰り返す。
- 停止ルール（ここに当たったら、その指示で止めて残りは未実行のまま温存し、ユーザーに報告する）:
  1. 指示に「実行後に一旦停止（ユーザー確認待ち）」と書かれている。
  2. `npm run lint` / `npm run build` が通らず自力で直せない。
  3. 指示内で判断できない点がある（勝手に決めない。「確認が必要」として止める）。
- 各指示ファイルの「完了条件（自己検証）」は必ず全て満たす。満たせない指示は「完了」にしない。

## セッション終了時に必ずやること

1. `docs/sessions/YYYY-MM-DD.md` を作成（当日の日付）
2. 以下のテンプレートに沿って記録する

## ChatGPT との並走（scripts/ask-chatgpt.mjs）

設計判断・コードレビュー・方針検討などで ChatGPT の意見も取り入れたいときは以下を使う。

### 前提: `.env.local` に API キーを追加
```
OPENAI_API_KEY=sk-...
```

### 使い方（Claude が Bash ツールで呼び出す）
```bash
# 質問を直接渡す
node scripts/ask-chatgpt.mjs "ここに質問"

# Markdown ファイルを渡す（長い仕様・コードレビュー向け）
node scripts/ask-chatgpt.mjs --file docs/sessions/review-request.md
```

### いつ使うか
- 設計の方向性について第二意見が欲しいとき
- 実装したコードを別視点でレビューしてほしいとき
- Claude が提案した内容をユーザーが ChatGPT にも確認したいとき

Claude はスクリプトの結果をそのまま貼り付け、内容をユーザーと一緒に検討する。

## プロジェクト概要（常時参照）

- **目的**: LUXAS 単一店舗向け予約・顧客管理の内部プロトタイプ（v0.1）
- **スタック**: Next.js / TypeScript / Tailwind CSS / Supabase / Vercel
- **データ保存**: 現在は localStorage（モックデータ）。Supabase 移行は v0.2 以降
- **ドキュメント**: `docs/` フォルダに仕様・設計・スキーマあり
- **開発コマンド**: `npm run dev` / `npm run lint` / `npm run build`
