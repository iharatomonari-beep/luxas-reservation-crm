# LUXAS 予約・顧客管理システム 社内レビュー ご案内

このシステムは現在「プロトタイプ（試作版）」です。実際の使い勝手・画面・流れを確認し、改善点を教えてください。

---

## 1. アクセス方法

- URL: `https://luxas-reservation-crm.vercel.app/`
- 開くと自動的にログイン画面へ進みます。**右下の「管理画面プレビューを開く」ボタン**を押すと、ログイン不要で管理画面に入れます（試作版のため）。
  - もしボタンが出ない/入れない場合は、URL末尾に `/dashboard` を付けて開いてください（例: `.../dashboard`）。
- スマホでも確認できます（同じURL）。

### 公開オンライン予約ページ（お客様が予約する画面）
- 管理画面の**上部バー右側「↗ 予約ページ」ボタン**から開けます（選択中の店舗の予約ページが新しいタブで開きます）。
- 直接URL: `https://luxas-reservation-crm.vercel.app/book/store-shibuya`（渋谷の例）

## 3. いまは「未完成・仮」の部分（※ここは指摘不要です）

レビューの手間を減らすため、現時点で**わざと未完成にしている**部分を共有します。ここは報告いただかなくて大丈夫です。

- **ログインは仮**（メール/パスワードの本認証は未実装。プレビューボタンで入る方式）。
- **メール送信は行われません**（確認メール等は画面表示のみのモック）。
- **写真はプレースホルダ**（「写真」と表示される枠。実画像は後で差し替え）。
- **会計・売上の数字はサンプルデータ**（実績ではありません）。
- **データは各自のブラウザに保存**されます（サーバー未接続）。そのため、他の人の入力は見えません／別端末では別データになります。
- ソーシャルログイン・決済・回数券・LINE連携などは未実装。

## 4. フィードバックの書き方（このまま箇条書きでOK）

各項目に、分かる範囲で次を添えてください：

```
- 画面：（例 公開予約ページ／予約台帳／マイページ など）
- 内容：（何が変か・どうしたいか）
- 種類：（バグ／文言／レイアウト／使いにくい／要望）
- できればスクショ
```

例：
- 画面：公開予約ページ／内容：コースの価格が小さくて見づらい→大きく／種類：レイアウト
- 画面：予約台帳／内容：今日の日付が分かりにくい／種類：使いにくい

## 5. 困ったとき

- **表示が崩れた／変なデータになった**：ブラウザの再読み込み（Cmd/Ctrl+Shift+R）。
- **データを最初に戻したい**：そのページで「データ初期化」系の操作、または開発担当に連絡（ブラウザの保存データをリセットします）。
- **入れない**：URL直 `/dashboard`、それでもダメなら開発担当へスクショ付きで連絡。
