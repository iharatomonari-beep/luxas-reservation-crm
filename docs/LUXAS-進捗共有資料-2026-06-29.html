各ステップの中身を、もう少し詳しくご説明します。

ステップ1：画面・機能をつくる　✓ ほぼ完成

お店の業務に必要な画面を、ほぼ一通り作りました。見た目も操作もほぼ完成しており、実際に使っている既存ソフト（PeakManager）に近い作りにしています。

  • 予約台帳 … 予約の登録・ドラッグでの移動・その場で会計まで
  • お客さま管理 … 来店履歴・カルテ・タグ・検索
  • スタッフ／メニュー／ブース／シフト の各管理
  • 会計・レジ・支払い（現金・クレジット・電子マネー）
  • 日次管理 … 出退勤・レジ金点検・売上日報・経費
  • 経営の数字 … 売上・客単価・スタッフ別／商品別の集計
  • メール（案内文・配信履歴）、各種設定、CSVの入出力

ステップ2：クラウドで共有・保存できるように　✓ 大半 完了

これまで“このパソコンの中だけ”に保存していたデータを、インターネット上の共有の保管庫へ移しました。これで複数の端末・スタッフが同じデータを扱えるようになります。

  • システムをクラウドに接続し、ログインを本物のセキュリティに（誰でも入れる状態を解消）
  • 約32種類のデータの置き場所をクラウドへ移設（予約・スタッフ・メニュー・売上・会計・日次・各種設定 など）
  • 「複数の端末で同じデータが見える」「再読み込みしても消えない」を1種類ずつ確認
  • お客さま情報を安全に扱う仕組み（操作の記録・権限・アクセス制限）も準備完了

→ “共有して保存できる仕組み”はほぼ完成しています。残るのは「本物の既存データを実際に入れること」で、これが次のステップ3にあたります。

① 本物のお客さま名簿の投入準備ほぼ完了・最終関門のみ

お客さまの情報は個人情報なので、いちばん慎重に扱う必要があります。そこで「本物の名簿を入れる前に、安全に扱える状態を作る」作業を、この間しっかり進めてきました。必要な安全対策8項目のうち6項目が完了しています。

これまでにやってきた作業（すべてテスト用の架空データで確認済み）
✓
① クラウドに「お客さま情報の置き場所」を用意
本物ではなく、架空のお客さま2件だけを入れて、正しく保存・表示できるか確認しました。
✓
② 「誰がいつ見た・直した・書き出した」を記録するしくみを設置
操作の記録（ログ）を残します。ただし記録の中に氏名・電話・カルテ内容などの個人情報そのものは残さない設計で、漏れないことも確認済みです。
✓
③ お客さま名簿の「書き出し（CSV）」を責任者だけに制限
オーナー以外は書き出せません。画面のボタンを隠すだけでなく、サーバー側でも止めるので、回り道してもデータを取り出せません。
✓
④ アクセス制限を本番レベルに強化
どのデータも「正しい権限を持つ人だけ」が扱える状態にしました。予約の二重登録を防ぐしくみも実際に効いていることを確認。
✓
⑤ 名簿の「取り込み」を架空データでテスト
取り込みの件数、重複チェック、項目の入り方まで、すべて正常に動くことを確認しました（本物の名簿はまだ入れていません）。
✓
⑥ バックアップ（万一の復元）の手順書を作成
もしもの時にデータを元に戻すための手順書を用意しました。
残っているのは、本物の名簿を入れる前の「2つの関門」だけ（社長・担当の作業）
⑦ バックアップを有効化して、復元テストを1回行う ／ ⑧ 個人情報の扱いを外部の専門家に確認してもらう
この2つが済んでから、本物のお客さま名簿を投入します。

お客さま情報の取り込み — 今後の進め方関門クリア後に実施

2つの関門（バックアップ・専門家確認）をクリアしたあと、本物のお客さま名簿は下の流れで取り込みます。取り込みの仕組み自体はすでに完成・検証済みなので、手順に沿って進めるだけです。

本番の取り込み手順
1
既存ソフト（PeakManager）からお客さま名簿を書き出す（CSV）
オーナーの作業。これまでの名簿を1つのファイルとして取り出します。
2
まず少量（数件〜数十件）で「試し取り込み」
いきなり全件ではなく、少しだけ入れて中身を確認できます。問題があってもすぐ戻せる安全な進め方です。
3
問題なければ全件を取り込み → クラウドに保存
取り込み画面からファイルを読み込むだけ。同じお客さまを二重に登録しないよう、重複は自動でチェックします。
4
取り込み後の確認
件数・氏名・来店履歴・カルテなどが正しく入っているかを確認します。誰がいつ何件入れたかは記録されます（個人情報そのものは記録に残しません）。
✓
日常運用へ
複数の端末・スタッフで、同じお客さま情報を共有して使えるようになります。

取り込まれる主な項目

  • 氏名・フリガナ・電話・メール・住所・生年月日
  • 会員番号・来店履歴（初回／最終来店・来店回数・売上）
  • カルテメモ・注意事項・タグ など

既存ソフト（PeakManager）独自の項目も、できる限り保持して取り込みます。

⚠ パスワードについて（要・お客さま案内）

お客さまのログインパスワードは引き継げません（どのシステムも暗号化して保存しており、元の文字を取り出せない仕様のため）。オンライン予約のログインを使う場合は、お客さまが初回にご自身で設定し直す形になります。これはどのシステムでも共通の一般的な仕様です。

安心ポイント … この取り込みの仕組みは、すでにテスト用の架空データで「件数・重複チェック・項目の入り方」まで動作を確認済みです。そのため本番の取り込みも、大きなつまずきなく進められる見込みです。

② 本物の既存データ（予約・スタッフ・メニュー等）の取り込み受け皿は完成・取り込みはこれから

こちらは、お客さま情報以外のお店の業務データ全般の話です。これまで“このパソコンの中だけ”に保存していたデータを、みんなで共有できるクラウドへ移す作業を、種類ごとに順番に進めてきました。

✓

「作った予約が再読み込みで消える」現象がありましたが、原因（通常予約に“部屋”を必須にしていた設定）を特定して解決しました。

残っているのは「本物の既存データの取り込み」
実際のスタッフ・メニュー（約375件）・過去の予約などを、完成した受け皿に取り込む作業です。
取り込みの“道すじ”はテスト用データで実証済みなので、本番の取り込みはスムーズに行える見込みです。
