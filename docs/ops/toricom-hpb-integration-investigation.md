# ④ TORICOM（HPB予約取込）連携 — 調査と判断（提案のみ・STOP）

最終更新: 2026-06-30 / 安全レベル: **STOP（外部依存・実装着手しない）**

この文書は LUXAS に「ホットペッパービューティー（HPB）の予約を取り込む」ための連携方式を調査し、次アクションを示すもの。実装には進まない（外部事業者の可否確認が前提のため）。

## 1. わかったこと（一次情報ベース）

TORICOM は **PeakManager（提供: EPARK, Inc.）のオプション**で、HPB / SALON BOARD と PeakManager の予約を**自動連動**させるサービス。要点:

- 自動連動する情報: **予約情報・シフト情報・顧客情報**（※顧客情報は SALON BOARD → PeakManager の一方向のみ）。
- HPB からの予約は「予約情報・メニュー・顧客名（フリガナ）・電話番号」まで取得して PeakManager に自動反映する。
- 直接入力（電話・飛び込み・施術後予約）は **PeakManager 側で入力**することで連動。SALON BOARD への入力は反映されない。
- 仕組みは「各予約サイトから自動取得し、自動反映させる」中継型（スクレイピング/RPA的）。HPB の公開APIではない。
- 他社の連携製品（例: SalonConnect）とは併用不可（TORICOM 利用時は併用機能が停止される、という第三者FAQあり）。

→ **重要**: TORICOM は EPARK / PeakManager に閉じた連携であり、**LUXAS のような外部CRMが直接叩ける公開API / Webhook / CSV出力は確認できない**。データの実体は PeakManager 内に入る。

## 2. 連携方式の候補と評価（LUXAS視点）

| 方式 | 概要 | 実現性 | 主な確認先 |
|---|---|---|---|
| A. PeakManager 経由で取得 | TORICOM が HPB予約を PeakManager に集約しているので、**PeakManager 側のエクスポート / API / CSV** を使って LUXAS へ取り込む | 中（PMの外部出力可否しだい） | EPARK / PeakManager |
| B. SALON BOARD の予約CSVを手動/定期取込 | HPB管理画面（SALON BOARD）から予約をCSV出力し LUXAS の取込画面へ | 高（運用で回る・最小実装） | HPB(リクルート)規約 |
| C. TORICOM から LUXAS へ直接連携 | TORICOM が外部CRMへ配信（API/Webhook） | 低（公開IFが確認できない） | EPARK / TORICOM |

現実解は **A か B**。C は期待しない（要EPARK確認）。

## 3. 本筋＝第三者提供の可否確認（先に潰す論点）

HPB予約には**顧客の個人情報（氏名・フリガナ・電話番号）**が含まれる。これを EPARK/HPB の外（LUXAS）へ流すには、技術以前に契約・規約上の**第三者提供の可否**を確認する必要がある。

- 御社（株式会社東邦）→ EPARK / リクルート（HPB）へ、**LUXASへのデータ提供・取得が規約上許されるか**を確認するのが本筋。
- これは §5 #8（個人情報の専門家確認）と直結する。実顧客データを扱うため、#6 バックアップ・#8 専門家確認が済むまで実取込はしない。

## 4. EPARK / PeakManager へ送る質問（そのまま使える）

1. PeakManager に蓄積された予約（HPB由来含む）を、**外部システムへ出力する手段**（API / Webhook / 定期CSV）はあるか。あれば仕様・認証方式・項目・更新頻度を知りたい。
2. その出力に**顧客の氏名・電話番号等の個人情報**を含められるか。含める場合の**第三者提供の取り扱い**（同意・契約上の条件）はどうなるか。
3. TORICOM 自体に、PeakManager 以外の外部CRMへ連携する機能はあるか。
4. 出力の**識別子**（予約ID・顧客ID等）は安定して再取得・突合可能か（重複・更新の判定に使えるか）。
5. 併用制限（他連携サービスとの排他）や、現行のLUXAS運用への影響はあるか。

## 5. 結論と次アクション

- **結論: STOP。** 方式（A/B/C）と第三者提供の可否が確定するまで LUXAS 側は実装しない。
- 次アクション（オーナー作業）:
  1. EPARK / PeakManager に §4 の質問を送る（方式Aの可否確認）。
  2. 並行して、HPB(SALON BOARD) の予約CSV出力可否・項目を確認（方式Bの当たり付け）。
  3. 第三者提供の可否を §5 #8（専門家確認）の枠で確認。
- 確認が取れたら、LUXAS 側は既存の **CSV取込経路（import-export）を流用した方式B** から最小実装するのが安全（既に架空CSV→DB取込はS5で実証済み）。方式Aが可能なら定期取得を別途設計。
