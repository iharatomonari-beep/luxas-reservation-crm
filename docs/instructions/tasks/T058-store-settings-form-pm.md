# タスクID：T058
# タスク名：店舗設定をPMの店舗マスタフォームに拡張（全フィールド＋アコーディオン）

## 安全レベル
CAUTION（店舗設定フォーム拡張。外部連携/認証/決済はSTOP）

## 目的
PMの店舗マスタ詳細フォームに合わせ、LUXASの店舗設定を拡張する。

## PMの仕様
- 基本: 企業ID/エリアID/店舗ID/店舗コード/店舗名*/略称*/郵便番号/都道府県*/市区町村番地*/住所2/電話/FAX/メール/HP URL/担当部署/担当者名/背景色設定。
- アコーディオン: 業務設定 / 会員番号の発行処理 / 予約用ホームページ / オンライン予約・店舗通知 / 店舗サマリ / 利用可能クレジットカード会社一覧 / 利用可能電子マネー一覧 / ウィジェット / インボイス設定。

## 対象ファイル
- features/store-settings/store-settings-manager.tsx（既存の営業時間/きざみは維持して拡張）
- 型: StoreSettings に任意フィールド追加（住所/電話/メール/HP/担当/背景色 等）

## 実行内容
- 基本フィールドを追加（既存の営業時間・きざみは残す）。
- アコーディオンセクションを用意（中身は最小：表示・保存できる項目のみ。外部連携実装はしない）。
- クレカ/電子マネー一覧セクションは既存マスタの一覧表示でよい（参照）。
- 保存はlocalStorage（既存方式）。

## 変更してはいけないファイル
- 認証/決済/外部連携の実装（STOP）。台帳が参照する営業時間キーは壊さない。

## 検証
- npm run lint / npm run build 成功。実機で各フィールド入力・保存、台帳の営業時間連動が従来どおり。

## 完了条件
1. 基本フィールド＋アコーディオン。2. 営業時間連動維持。3. lint/build OK。4. _index更新＋sessions追記。5. 実機OKで完了。

## 停止条件
- 外部連携/決済/認証が要るなら STOP。台帳連動が壊れるなら停止。

## 報告
- 完了ID T058 / 変更ファイル / lint・build / 実機確認依頼。
