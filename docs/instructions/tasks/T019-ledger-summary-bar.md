# タスク名：予約台帳PM準拠 作り込み②＝予約集計バー（当日集計＋支払方法別内訳）

## 安全レベル
CAUTION（予約台帳のUI/集計ロジック追加。実行前にユーザー確認）

## 前提
T018 完了後に着手（同一ファイル衝突回避）。未完了なら保留・停止。

## 目的
タイムライン最下部に、PMと同様の予約集計バーを置く。当日の予約データ(localStorage)から計算できる項目は実値、未実装の売上/支払系は0またはダミーで枠だけ用意する。

## 参照（PMの集計バー項目・catalog §2）
- 1段目: 目標 / 総販売額 / 新規 / リピート / 総来店人数 / 施術件数 / 客単価 / 取消 / 無断キャンセル / 返客
- 2段目: 施術 / 物販 / 回数券販売 / プリベイド販売（各 実績/目標）
- 3段目: 現金 / クレジット / 回数券利用 / プリベイド利用 / ポイント利用 / 商品券利用 / 電子マネー / EPARKサービス

## 実行内容
1. 既存の簡易バッジ（予約中N等）を、PM項目に沿った集計バーへ拡張する。
2. 当日の予約配列から計算: 施術件数(=当日有効予約数) / 総来店人数 / 新規・リピート(顧客の判定基準が無ければ件数のみ＋「要確認」) / 取消・無断キャンセル(status=canceled等から) / 客単価(売上が無ければ「-」)。
3. 売上・支払方法別(現金/クレジット/電子マネー/回数券/プリベイド/ポイント/商品券/EPARK)は、LUXASに会計データが無いため **0/ダミー表示＋「会計未実装」の注記**。目標は店舗設定/月間目標が無ければ0。
4. 算出基準が曖昧な項目（新規/リピートの定義、客単価の元）は勝手に決めず「要確認」として注記。
