# タスクID：T009
# タスク名：タイムラインでシフト時間を色分け（シフト内=白／シフト外=薄いグレー）

## 安全レベル
CAUTION（予約台帳のUI変更。実行前にユーザー確認）

## 目的
各スタッフ行のタイムラインで、勤務シフトが一目で分かるように背景を色分けし、シフト外は予約を作れないようにする。
- シフト時間内 = 白（クリックで予約作成可）
- シフト時間外 = 薄いグレー、かつ **クリック不可**（空き枠クリックで予約作成画面を開かない）
PeakManager と同様に「シフトの時間が分かり、シフト外がグレーで触れない」状態にする。

## 背景
- お手本: PeakManager の予約台帳（シフト時間を表示、シフト外はグレー）。スクショ `~/Desktop/pm_スクリーンショット/スクリーンショット 2026-05-16 15.29.08.png` の「シフト」表示も参考。
- スタッフ別の色分けは **しない**（全スタッフ同じ配色）。
- 表示日のシフトは既存ロジックで取得済み（`getVisibleStaffForSelectedDate` / `findShiftForReservation` / `shifts` データ、`features/reservations/reservation-ledger.tsx`）。
- 休憩時間は T006 で空にしたため、今回は「シフト内／シフト外」の2色だけでよい。

## 実行内容
1. 各スタッフ行のタイムライン背景を、その日のシフト（`startTime`〜`endTime`）に基づいて塗り分ける:
   - シフト時間内のセル/領域 = 白（例 `bg-white`）。
   - シフト時間外のセル/領域 = 薄いグレー（例 `bg-stone-100` 程度の控えめなグレー。既存の配色トーンに合わせる）。
2. 全スタッフ同じ配色にする（スタッフごとに色を変えない）。
3. **シフト外のセルはクリック不可**にする: 空き枠クリックで予約作成画面を開かない（ホバーの「＋」も出さない）。シフト内のセルは従来通りクリックで予約作成。
4. 予約カード・5分グリッド線・時刻ヘッダーの表示自体は変えない。
4. グレーの濃さや境界の見せ方で迷う場合は、控えめ（薄い）側に倒し、判断に迷う装飾は「要確認」として残す。
