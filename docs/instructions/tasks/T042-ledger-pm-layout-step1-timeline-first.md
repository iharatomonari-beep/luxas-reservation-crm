# タスクID：T042
# タスク名：予約台帳をPM配置に寄せる Step1（タイムラインを最上部の主役に・縦の流れをPM化）

## 安全レベル
CAUTION（`reservation-ledger.tsx` 1ファイル内のレイアウト再配置。ロジック変更なし）

## 目的
予約台帳の画面構成を PeakManager（PM）の `/Orders` に忠実に寄せる第1段階。
現状は上部に「タイトル＋説明文／サマリカード3枚／当日情報3タブ／選択中の予約／保留棚」が積まれ、**タイムラインが下に押し下げられている**。PMは「上部のコンパクトな操作バー → すぐタイムライン（主役）」の順。
このタスクでは **JSXの並べ替えと一部削除のみ** を行い、タイムラインを最上部の主役にする。**機能・ロジックは一切変えない。**

## 背景（PM実機の配置・お手本）
PM `/Orders` 上から: ①コンパクトな日付操作バー → ②表示切替タブ＋ドラッグ用バー → ③（左に縦アクションレール＋）タイムライン → ④下部ツールバー → ⑤予約集計サマリ。
※ ②の5タブ化・ドラッグ用横長バー・集計バーの下移動・上部バーの天気/すべて/店舗/時計は **後続タスク（T043/T044）** で行う。本タスクは触らない。

## 対象ファイル / 変更してよいファイル
- `features/reservations/reservation-ledger.tsx` のみ

## 変更してはいけないファイル

## 実行内容（並べ替え・削除のみ。ロジック不変）
現状の最上位 `return ( <div className="space-y-4"> … )` 内の並び順を、次のように変更する。

1. **タイトル群を削除**
   - ヘッダー section 内の「`Reservation Ledger`」ラベル（`<p class=...>Reservation Ledger</p>`）、`<h1>予約台帳</h1>`、その下の説明文 `<p>電話予約を受けながら…まとめて確認できます。</p>` の3要素を削除する。
   - **「新規予約」ボタンと日付ナビ（前日／今日へ戻る／日付input／翌日）は残す。** これらを横1行のコンパクトなバーとして上部に置く（左に日付ナビ、右に新規予約、もしくは現状の並びのままで可。見た目の最終調整はT044）。

2. **サマリカード3枚を削除**
   - `<section className="grid gap-2 sm:grid-cols-3">` の中の `表示日` / `予約数` / `担当数` の `SummaryCard` 3枚を含む section を**まるごと削除**する。
   - （注）`SummaryCard` コンポーネント自体は他で使っていなければ未使用警告が出る可能性。出たら import とコンポーネント定義は残してよいが、未使用 import 警告が lint を落とす場合のみ該当 import を外す（それ以外は触らない）。

3. **3つのパネルをタイムラインの「下」へ移動**
   - 次の3 section を、現在の位置（タイムラインより上）から、**タイムラインブロックの直後**へ移動する。順序は「当日情報 → 選択中 → 保留棚」を維持。
     - (a) 当日情報パネル（コメント `当日情報パネル（PM準拠・T036）` が付いた `<section>`。予約情報/販売情報/返客情報の3タブ）
     - (b) 選択中の予約 section（`{selectedReservation ? ( <section …>選択中の予約… )} : null}` のブロックごと）
     - (c) 保留棚 section（`{heldReservations.length > 0 && ( <section …>保留棚… )}` のブロックごと）
   - 「タイムラインブロック」とは `<div className="flex items-stretch gap-3">`（左縦アクションレール＋タイムライン本体）から、その対になる `</div>` までの一塊。**この塊は中身を変えずそのまま**にし、(a)(b)(c) をこの塊の**直後**（`ReservationDetailModal` の直前）へ移す。

4. 最終的な並び（return直下）:
   1) ヘッダー（新規予約＋日付ナビのコンパクト行）
   2) `StatusMessage`（現状のまま）
   3) Debug section（現状のまま・`isDebugMode` 条件付き）
   4) タイムラインブロック（左レール＋タイムライン本体＝中身不変）
   5) 当日情報パネル（移動）
   6) 選択中の予約（移動）
   7) 保留棚（移動）
   8) 各モーダル（現状のまま）

## してはいけないこと
- タイムライン本体・予約カード・ドラッグ処理・予約集計バー・下部ツールバーの**中身は変更しない**（位置も今回は動かさない＝集計バーの下移動はT043）。
- 表示切替タブ（基本/全体）の増設はしない（T043）。
- 天気/すべて/店舗セレクタ/時計の追加はしない（T044）。
- state・ハンドラ・計算ロジック・型は一切変更しない。純粋にJSXの移動と削除のみ。
