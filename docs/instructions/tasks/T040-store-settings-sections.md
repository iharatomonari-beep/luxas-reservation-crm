# タスクID：T040
# タスク名：店舗設定の拡張セクション（業務設定/会員番号発行/予約用HP/オンライン予約・店舗通知/インボイス）

## 安全レベル
CAUTION（設定画面の拡張）。**外部連携・決済・認証には踏み込まない（STOP対象）**。

## 目的
PM店舗マスタの折りたたみセクション相当を、LUXASの店舗設定(/dashboard/settings)に追加する。各種の店舗運用設定をローカルに保持。

## 背景・注意
- catalog §「店舗マスタ」の折りたたみ: 業務設定 / 会員番号の発行処理 / 予約用ホームページ / オンライン予約・店舗通知 / (店舗サマリ/クレカ一覧/電子マネー一覧/ウィジェット/インボイス)。**各セクション内部は未巡回**＝PM標準で設計、相違あれば実機再確認。
- オンライン予約の実連携・通知の実送信・決済はしない（枠と設定値の保持のみ）。

## 対象ファイル / 変更してよいファイル
- `app/dashboard/settings/page.tsx`（T014の画面を拡張）
- `features/store-settings/*`、`features/master-data/`の店舗設定型（StoreSettings拡張）

## 変更してはいけないファイル
- `lib/supabase/` `supabase/` `.env*` `package.json`、認証(`components/auth/`,`app/login`)＝STOP
- `features/reservations/`等の既存ロジック、上記以外

## 実行内容
1. 設定をセクション折りたたみで整理:
   - 業務設定: 営業時間(T031)/予約単位slotMinutes/予約受付開始終了。
   - 会員番号の発行処理: 採番ルール(接頭辞/桁/次番)を保持（実発行は顧客作成時に利用、最小で）。
   - 予約用ホームページ: 店舗説明/画像URL/営業時間/定休日（表示用テキストの保持）。
   - オンライン予約・店舗通知: 公開ON/OFF・通知先などの設定値保持（実連携なし）。
   - インボイス設定: 登録番号など保持。
2. すべてStoreSettings（localStorage）に保持。実連携/送信/決済は行わない。

## 検証方法
- `npm run lint` / `npm run build`／実機でユーザー確認。

## 完了条件
1. 各セクションの設定を保存・再表示できる。2. 既存(営業時間連動T031)を壊さない。3. lint/build OK。4. sessions追記・_index更新。5. 実機OKで完了。

## 停止条件
- オンライン予約の実連携/通知実送信/決済/認証に踏み込む必要が出たらSTOP。会員採番の実装範囲は推奨(設定保持＋作成時利用)で進め報告。指定外ファイルはSTOP。lint/build失敗で停止。

## 完了後の報告内容
- 完了ID/追加セクション/保持項目/実連携しない範囲/要実機再確認点/lint・build/_index更新/実機確認依頼
