# タスクID：T054
# タスク名：タグを3画面に分離（顧客タグ／予約ルートタグ／施術カルテタグ）＋管理コード

## 安全レベル
CAUTION（既存tag-managerの再編。データ構造は種別フィールドで分離）

## 目的
PMはタグを3つの独立画面で管理し、予約ルートタグ・施術カルテタグには「管理コード」がある。LUXASは1画面統合。これを3画面に分ける。

## PMの仕様
- 顧客タグ: ID / 名前 / 表示順序
- 予約ルートタグ（プロモーション）: ID / 名前 / 管理コード / 表示順序
- 施術カルテタグ: ID / 名前 / 管理コード / 表示順序

## 対象ファイル
- features/master-data/tag-manager.tsx（種別ごとに分離 or 種別固定で3ルート）
- app/dashboard/customer-tags/page.tsx, route-tags/page.tsx, chart-tags/page.tsx（新規 or 既存tagsを種別パラメータで）
- components/layout/top-menu.tsx（顧客情報グループに3項目）

## 実行内容
- 既存のタグ種別（顧客/予約ルート/カルテ）のデータを活かしつつ、画面を3つに分ける（既存の種別切替UIを廃し、各画面は1種別固定）。
- 予約ルート・カルテに「管理コード」フィールドを追加（型に code?:string）。
- 上部メニューを3項目に。

## 変更してはいけないファイル
- 既存タグデータの破壊変更（種別の持ち方は維持）。予約受付の予約タグ参照を壊さない。

## 検証
- npm run lint / npm run build 成功。実機: 3画面でCRUD、管理コード入力、メニュー3項目。

## 完了条件
1. 3画面分離＋管理コード。2. 既存タグ参照維持。3. lint/build OK。4. _index更新＋sessions追記。5. 実機OKで完了。
