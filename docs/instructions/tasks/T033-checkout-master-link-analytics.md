# タスクID：T033
# タスク名：会計の支払選択肢を決済マスタと結合＋経営指標/集計の実値化

## 安全レベル
CAUTION（会計UI・集計ロジックの結合）

## 目的
T022会計の支払方法選択肢を、T029のクレカ会社/電子マネーマスタから引く。会計データが入ることで、T019予約集計バー・T028経営指標の「ダミー」を実値に切り替える。

## 前提
T022(会計)・T029(決済マスタ)が実装済みであること。未了なら該当を先行（依存）として報告し止める。

## 対象ファイル / 変更してよいファイル
- `features/reservations/checkout-modal.tsx`（クレカ/電子マネーの選択肢をマスタ参照に）
- `features/reservations/reservation-ledger.tsx`（予約集計バーを会計実値に）
- `features/analytics/*`・`app/dashboard/page.tsx`（経営指標/トップKPIを会計実値に）

## 変更してはいけないファイル
- `features/customers/` `features/import-export/`
- `lib/supabase/` `supabase/` `.env*` `package.json` `tailwind.config.ts`、上記以外

## 実行内容
1. 会計モーダルのクレカ種類/電子マネー種類を、それぞれのマスタ(T029)の有効項目から選べるようにする。
2. 予約集計バー(T019)の 総販売額/支払方法別内訳/客単価 を、当日の会計済データから集計して実値表示。会計が無い日は0。
3. 経営指標(T028)/トップKPIの売上系を会計データ集計に切り替え（過去データが無い項目は0＋注記）。
4. 新規/リピートの定義: 当日予約のphone/customerNameが過去の会計済予約に在れば「リピート」、無ければ「新規」（推奨）。

## 検証方法
- `npm run lint` / `npm run build`／実機（会計→集計反映）でユーザー確認。

## 完了条件
1. 会計でマスタの支払種類を選べる。2. 集計バー/経営指標が会計実値で動く。3. 既存を壊さない。4. lint/build OK。5. sessions追記・_index更新。6. 実機OKで完了。

## 停止条件
- T022/T029未了なら依存先行として停止報告。新規/リピート定義は推奨で進め報告。指定外ファイルはSTOP。lint/build失敗で停止。
