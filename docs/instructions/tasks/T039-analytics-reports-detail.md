# タスク名：経営指標の詳細帳票（売上・明細集計/クレジット会社別/日次集計/帳票出力）

## 安全レベル
CAUTION（集計画面の追加）

## 前提
会計データ(T022)＋決済マスタ結合(T033)があると実値化できる。無い項目は0/ダミー＋注記で可。

## 目的
T028で作った簡易集計に加え、PMの経営指標の各帳票を追加する。

## 背景・注意
- catalog §E（売上・明細集計/店舗別/スタッフ別/商品別/物販/時間帯別来店/帳票出力/日次集計(日別・個人別)/クレジット会社別/店舗集計/顧客数推移）。**各帳票の実機内部は未巡回**＝PM標準＋会計データで設計。相違あれば実機再確認。
- 単一店舗のため「店舗別/店舗集計」は対象外（または当店のみ）。

## 対象ファイル / 変更してよいファイル
- `features/analytics/*`、`app/dashboard/analytics/*`、`components/layout/sidebar.tsx`

## 変更してはいけないファイル
- `package.json`（グラフlib追加=STOP。簡易テーブル/バーで）
- `lib/supabase/` `supabase/` `.env*`、`features/reservations/`等の既存ロジック、上記以外

## 実行内容（1つずつ・テーブル中心）
1. 売上・明細集計: 期間指定で 売上/件数/客単価 の集計テーブル＋明細。
2. クレジット会社別: 会計のクレカ種類別 売上集計。
3. 日次集計(日別): 日付別の 売上/来店/客単価。日次集計(個人別): スタッフ別×日。
4. 帳票出力: 集計のCSV書き出し（既存CSVユーティリティがあれば流用）。
5. 顧客数推移: 月別の新規/リピート/総数。
- 会計が無い段階は各値0＋「会計実装後に実値」と注記。
