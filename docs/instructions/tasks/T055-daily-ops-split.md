# タスクID：T055
# タスク名：日次管理を独立画面群に分割＋売上日報をPM全項目に拡張

## 安全レベル
CAUTION（既存の日次1画面4タブを独立ルート化。集計ロジックは流用）

## 目的
PMは日次管理を独立画面で持つ（開店処理/レジ金点検/閉店処理/閉店処理検索/売上日報/月間目標/出勤退勤）。LUXASは1画面4タブ。独立画面に分け、日報をPMの全セクションに拡張する。

## PMの仕様（要点）
- 開店処理/レジ金点検/閉店処理: 金種(10000〜1円)×本数×枚数×合計＋（登録/点検）ボタン。閉店は預け金プリフィル。
- 閉店処理検索: 出力日次範囲＋検索。
- 売上日報: 本日の結果/来店数/予算達成度/スタッフ提案数/明日の予定/実施プラン/振り返り/シフト状況/天気＋一時保存/日報送信。
- 月間目標入力: 日付範囲＋一覧（日付→日報編集）。
- 出勤・退勤: 当日シフト担当者＋出勤/退勤ボタン。

## 対象ファイル / 追加してよい
- 既存 features/daily/daily-ops.tsx を分割し、各機能を独立コンポーネント＋ルートへ:
  - app/dashboard/open-register/page.tsx（開店処理）
  - app/dashboard/check-register/page.tsx（レジ金点検）
  - app/dashboard/close-register/page.tsx（閉店処理）
  - app/dashboard/close-history/page.tsx（閉店処理検索）
  - app/dashboard/daily-report/page.tsx（売上日報・PM全項目）
  - app/dashboard/attendance/page.tsx（出勤・退勤）
  - 月間目標は既存 /dashboard/shifts/monthly の日次目標を流用 or /dashboard/monthly-target
- components/layout/top-menu.tsx（日次管理グループに各項目）

## 実行内容
- 既存の金種テーブル/会計集計/経費等のロジックを流用しつつ、画面を分ける。1画面ずつ実装・検証。
- 売上日報はPMの全セクションを表示（実値が無い項目はダミー/0＋注記）。一時保存/送信は保存のみ（送信はモック）。
- STOP: 実際の現金確定・締めを伴う破壊処理は作らない（入力・保存のみ）。

## 変更してはいけないファイル
- 会計・予約データの破壊変更。認証。

## 検証
- 各: npm run lint / npm run build 成功。実機で各独立画面が動く＋メニュー導線。

## 完了条件
1. 各機能が独立画面化。2. 日報がPM全項目。3. 既存集計維持。4. lint/build OK。5. 各完了で_index更新＋sessions追記。6. 実機OKで完了。

## 停止条件
- 1画面ずつ。壊れたら停止。締め/現金確定の実処理が必要なら停止して報告。指定外ファイルはSTOP。

## 報告
