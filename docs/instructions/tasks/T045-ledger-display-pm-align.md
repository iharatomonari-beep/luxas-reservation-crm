# タスクID：T045
# タスク名：予約台帳の「表示」をPMに合わせる（上部日付バー／5タブ／集計バーを下へ／ドラッグ帯）

## 安全レベル
CAUTION（reservation-ledger.tsx 1ファイル中心のレイアウト/表示変更。データ操作なし）

## 目的
予約台帳の見た目をPeakManager（PM）の /Orders に合わせる。PM実機で確認済みの差分を解消する。

## PM実機の該当表示（お手本）
- 上部に1行のコンパクトな操作バー: `<` 今日 `>`（前へ/今日/次へ）／日付表示／再読込アイコン／天気／（右）「すべて」絞り込み／店舗名／現在時刻。
- 表示切替タブ: 基本 / 全体 / シフト / ブース / 両方（5種）。
- 予約集計の数値は「タイムラインの下」（下部ツールバーの下）。
- タイムライン上部にドラッグ用の横長帯「日付をまたいで予定を変更したい場合は、一旦このエリアに置いてください」＋右に「シフト追加」。

## 対象ファイル / 変更してよいファイル
- features/reservations/reservation-ledger.tsx のみ

## 変更してはいけないファイル
- 上記以外すべて。

## 実行内容

### 1. 上部日付バーをPMコンパクト化
- 現在のヘッダー（新規予約ボタン＋前日／今日へ戻る／日付input／翌日）を、PM風の1行バーに整える。
  - 日付ナビ: `<`（前日・アイコンのみ）／今日（ラベル短縮）／`>`（翌日・アイコンのみ）／日付input。
  - 再読込ボタン（RotateCw等のアイコン）を追加。動作は最新データ再読込でよい（最小実装: location.reload() 可）。
  - 現在時刻表示（HH:mm）を右側に追加（useState＋useEffectで1分ごと更新・JST）。
  - 天気／「すべて」絞り込み／店舗セレクタは見た目だけの静的プレースホルダでよい（機能は後続）。
  - 「新規予約」ボタンは残す。

### 2. 表示切替タブを5種に
- 現在の「基本 / 全体」に「シフト / ブース / 両方」を追加し、PMと同じ5タブの見た目にする。
  - 基本・全体は既存どおり機能（timelineView "basic"/"full"）。
  - シフト・ブース・両方は実動作の実装はしない。見た目はタブとして並べ、title="準備中"の無効表示でよい（クリックで落ちないこと）。

### 3. 予約集計バーをタイムラインの下へ移動
- 現在タイムラインの「上」（ヘッダー直下）にある「予約集計バー」ブロック（コメント「予約集計バー（PM準拠・T019/T033）」の div 一塊）を、下部ツールバーの下（タイムライン section の末尾、閉じる直前）へ移動する。
  - 中身（数値・計算）は変更しない。位置だけ移す。border は見た目に合わせて border-t に調整可。

### 4. ドラッグ用横長帯＋「シフト追加」
- タイムライン上部（表示切替タブの下あたり）にPM風の横長帯を追加: 「日付をまたいで予定を変更したい場合は、一旦このエリアに置いてください」。
  - 既存の保留棚（holdShelf）のドロップ先として機能させるのが理想。ドラッグ結線が重ければ、見た目の帯＋保留棚への誘導説明に留めてよい（既存の保留棚セクションは残す）。落ちないことを優先。
  - 右端に「シフト追加」ボタン（リンク先 /dashboard/shifts/monthly）。
  - やり過ぎず見た目をPMに寄せることを優先。難しければ説明帯＋シフト追加リンクのみで可（その旨を報告）。

## してはいけないこと
