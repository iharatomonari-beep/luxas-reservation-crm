# タスクID：T007
# タスク名：予約台帳タイムラインに「基本（5時間）」「全体（終日）」の表示切替を追加

## 安全レベル
CAUTION（予約台帳のUI/挙動変更。実行前にユーザー確認）

## 目的
PeakManager の予約台帳と同様に、タイムラインの表示を「基本」と「全体」の2種類で切り替えられるようにする。
- 基本: 5時間分を表示（通常スケール）。
- 全体: 営業時間すべて（終日）を表示。

## 背景
- お手本スクショ: `~/Desktop/pm_スクリーンショット/スクリーンショット 2026-05-16 15.29.08.png`（上部に「基本 / 全体 / シフト / ブース / 両方」タブ。今回は **基本・全体の2つだけ** 作る）。
- 現状のタイムラインは営業時間全体（例 10:00-23:00）を横スクロールで表示している（＝実質「全体」に近い）。
- 営業時間・予約単位は店舗設定（T002 で導入済みの `StoreSettings`）から取得する想定。

## 対象ファイル
- `features/reservations/reservation-ledger.tsx`（読む・変更する）
- 参照のみ: 店舗設定（T002 で追加したファイル。`businessStartTime`/`businessEndTime`/`slotMinutes`）

## 変更してよいファイル
- `features/reservations/reservation-ledger.tsx` のみ

## 変更してはいけないファイル
- 店舗設定ファイル（読むだけ。値は変えない）
- `features/reservations/reservation-create-page.tsx`
- `features/customers/`、`features/import-export/`、`features/master-data/`
- `lib/supabase/`、`supabase/`、`app/`、`.env` / `.env.local`、`package.json`、`tailwind.config.ts`
- 上記「変更してよいファイル」以外すべて

## 実行内容
1. タイムライン上部に「基本 / 全体」の切替（タブまたはトグル）を追加する。初期選択は「基本」。
2. 「基本」表示:
   - 表示する時間範囲を **5時間分** にする。
   - 初期の開始位置は **現在時刻** を起点に5時間（例 13:20 なら 13:20〜18:20 付近を表示）。横スクロールで営業時間内の前後も見られるようにする。
   - 現在時刻が営業時間外（営業開始前・終了後）の場合は、**営業開始時刻から5時間**にフォールバックする。
   - 5分グリッド・カード配置のスケールは現状（通常）を維持。
3. 「全体」表示:
   - **営業時間すべて**（`businessStartTime`〜`businessEndTime`）を表示する。
   - 終日が見渡せること（現状の全幅スクロール表示でよい。可能なら画面幅に収める）。
4. 切替時に既存の予約カード位置・ドラッグ・クリック等が壊れないこと。
5. 「5時間の起点（営業開始 or 現在時刻）」「全体を画面幅に収めるか／スクロールのままか」で迷う場合は、勝手に決めず停止して確認する。
