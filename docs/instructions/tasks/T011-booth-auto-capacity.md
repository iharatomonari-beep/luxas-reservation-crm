# タスクID：T011
# タスク名：予約のブース固定を撤去し、容量ベース判定に作り直す

## 安全レベル
CAUTION（予約作成・予約台帳の主要ロジック変更。実行前にユーザー確認）

## 目的
予約に個別ブースを固定する方式をやめ、ブースは自動配置（種別ごとの台数で空き判定）にする。これにより、ドラッグ移動で「同じブースの予約と重複しています」が出て動かせない問題を解消する。

## 前提（依存）
T010（個室必須フラグ＋容量判定ヘルパ）が完了していること。未完了なら実行せず「T010未完了のため保留」と報告して停止。

## 背景・確定仕様
- 予約作成・詳細から **個別ブース（ブースA/B…）の手動選択をなくす**。
- 予約詳細では **種別（個室 / 施術ブース）だけを表示**し、**自動・編集不可**にする。種別はメニューの `requiresPrivateRoom` から決まる（true=個室、false=施術ブース）。
- 個別ブース名は予約に固定しない。
- 空き判定（作成時・ドラッグ移動時とも）:
  - メニューが個室必須 → 同時刻に時間が重なる「個室予約」の件数が **個室台数（2）未満**ならOK、以上なら「個室の空きがありません」で不可。
  - そうでない → 同時刻の「施術ブース予約」の件数が **施術ブース台数（8）未満**ならOK、以上なら「施術ブースの空きがありません」で不可。
  - 判定は T010 で作った容量ヘルパを使う。
- スタッフの重複（同一スタッフ・同時間）チェックは **これまで通り残す**（今回外すのはブース固定だけ）。
- シフト外チェックも残す。休憩チェックは T006 で実質無効（休憩空）なのでそのまま。

## 実行内容
1. 予約作成（`reservation-create-page.tsx`）:
   - ブース選択（ブースA/B…の手動選択）を撤去する。
   - 代わりに、選択メニューから決まる **種別（個室 / 施術ブース）を読み取り専用で表示**する。
   - 保存時に個別ブースを固定しない（`roomId` を必須にしない）。
2. 予約可否判定（作成・ドラッグ移動）:
   - 既存の「同一ブース（roomId）重複でブロック」を **削除** し、T010 の容量ヘルパによる「種別ごとの台数判定」に置き換える。
   - スタッフ重複・シフト外チェックは残す。
3. 予約台帳（`reservation-ledger.tsx`）:
   - 予約詳細・カードの「ブース」表示を、個別ブース名ではなく **種別（個室/施術ブース）** にする。
   - ドラッグ移動時、移動先で容量超過なら「（種別）の空きがありません」で元に戻す。
4. `Reservation.roomId` を完全に消すかは影響が大きい。**まずは roomId を任意（optional）扱いにして判定に使わない方向**で進め、型から削除する必要が出たら停止して確認する。
