# タスクファイル作成ルール（docs/instructions/tasks/）

このフォルダには、Claude Code に実行させる**個別タスクファイル**を置く。
実行する側（Claude Code）は、まず `docs/instructions/_execution-policy.md`（常設ルール）→ `docs/instructions/_index.md`（実行キュー）を読み、そのうえで各タスクファイルを読む。

---

## タスク作成ルール

- 1ファイルにつき1タスクにする。
- タスクIDは `T001`, `T002`, `T003` … のように連番にする（既存の最大番号 + 1）。
- ファイル名は `T001-task-name.md` の形式にする（番号 + 短い英語スラッグ）。
- 各タスクに**安全レベルを必ず書く**（SAFE / CAUTION / STOP のどれか）。
- 対象ファイルと**変更禁止ファイル**を明記する。
- **完了条件**と**停止条件**を明記する。
- 曖昧な作業は CAUTION または STOP にする。
- **コード変更を含む作業は原則 CAUTION** にする。
- **DB・env・認証・削除・依存関係変更は STOP** にする。
- タスクファイルを作ったら、`docs/instructions/_index.md` の実行キュー表に行を追記する（状態の初期値は「未実行」）。

> 既存の `T001`〜`T003` は歴史的経緯で `docs/instructions/` 直下にある。**新規タスクはこの `tasks/` 配下に作成する。**

### 変更禁止ファイルとして常に意識する資産

`変更してはいけないファイル` には、最低限これらを保護対象として明記する。

- `features/customers/`（顧客管理）
- `features/import-export/`（CSV入出力）
- `lib/supabase/`、`supabase/`（DB・接続）
- `app/` 配下のレイアウト・認証
- `.env` / `.env.local`、`package.json`

「変更してよいファイル」に無いファイルを変更する必要が出たら、STOP（指示にないファイル変更）として停止する。

---

## タスクテンプレート

新規タスクは以下をコピーして使う。

```markdown
# タスクID：T0XX
# タスク名：
# 安全レベル：SAFE / CAUTION / STOP
# 目的：
# 背景：
# 対象ファイル：
# 変更してよいファイル：
# 変更してはいけないファイル：
# 実行内容：
# 検証方法：
# 完了条件：
# 停止条件：
# 完了後の報告内容：
```
