# タスクID：T005
# タスク名：予約台帳のドラッグ移動（時間変更・担当者変更）のUX仕上げ（PeakManager準拠）

## 安全レベル
CAUTION（予約台帳のUI/挙動変更。実行前にユーザー確認）

## 目的
予約台帳でカードをドラッグして「開始時刻の変更（横移動）」「担当スタッフの変更（縦移動）」を、PeakManager（お手本）と同等の操作感に仕上げる。ロジックは T001 で実装済みと確認済みなので、今回は**操作中の見せ方（フィードバック）と移動後の即時反映**を整えるのが主目的。大改修はしない。

## 背景
- お手本スクショ: `~/Desktop/pm_スクリーンショット/スクリーンショット 2026-05-16 15.29.08.png`（予約台帳タイムライン本体）、`…15.30.03.png`（予約受付メニュー）。
- ドラッグが「動いている動画」はこの画像フォルダに無いため、**操作中の見た目の細部は「要確認」**。最低限の受け入れ条件（下記）を満たせばよい。
- 仕様の正本: `docs/luxas-reservation-ledger-spec.md`（§5,6,13）、`docs/sessions/2026-06-13.md`（T001のドラッグ静的確認結果）。
- 前提: T004（Tailwind content 修正）が完了し、カードが時刻位置に正しく配置されること。未完了ならこのタスクは実行せず「T004未完了のため保留」と報告して停止。

## 実行内容
1. まず現状を確認して報告する（変更前）:
   - ドラッグ中に「ドロップ先のハイライト」や「移動中の時刻プレビュー（例 14:05）」が出るか。
   - 移動確定後に、下部の集計表示や「選択中の予約」パネルが即更新されるか。
   - 診断ログ `[drag:start]`/`[drag:move]`/`[drag:drop]` の有無（今回は削除しない。残す）。
2. 不足している場合のみ、次を満たすよう最小限で追加する（既存のドラッグ計算ロジックは変えない）:
   - ドラッグ中、ドロップ先のスタッフ行・時間帯が視覚的に分かるハイライトを出す。
   - ドラッグ中、移動後の開始時刻（5分スナップ後の値）をカード付近にプレビュー表示する。
   - 重複・シフト外でドロップした場合は、確定せず元位置に戻し、理由メッセージを出す（既存の検証を流用）。
   - 移動確定後、表示中の予約情報（選択パネル等）が即反映される。
3. 見た目の細部（色・角丸・影など）は PM の雰囲気に寄せるが、ピクセル一致は不要。判断に迷う装飾は「要確認」として残し、勝手に大きく作り込まない。

- `npm run lint`（エラー0）

- 実機（`npm run dev`）でユーザーが確認（下記「停止条件」の実機確認）。

## 完了条件
1. 現状確認の報告をした。
2. 不足分のみ最小追加し、lint / build が通った。
3. コード変更が `reservation-ledger.tsx` だけに収まっている。
4. `docs/sessions/2026-06-13.md` に変更点を追記し、`docs/instructions/_index.md` の T005 を更新した。
5. ユーザーの実機確認OKをもって「完了」とする（それまでは「実行中」または「保留」）。

## 停止条件
- T004 未完了なら停止（前提未達）。
- 実機確認が必要：ビルド通過後、ユーザーが次の5点を確認するまで「完了」にしない。
  1. 右ドラッグで開始時刻が5分単位でずれ、幅は変わらない。
  2. 別スタッフ行へドラッグで担当が変わる。
  3. 重複する位置へドロップでエラーになり元に戻る。
  4. ドラッグ中にドロップ先と移動後時刻が分かる。
  5. 移動後に表示が即更新される／リロードしても保持される。
- `reservation-ledger.tsx` 以外の変更が必要になったら STOP（指示外ファイル変更）。
- lint / build が通らず自力修正できない場合は停止。
