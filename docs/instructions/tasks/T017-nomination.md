# タスクID：T017（改訂版）
# タスク名：予約から「担当スタッフ必須」を撤去し、「スタッフ指名」ボタンに置き換える

> ⚠️ 改訂: 旧版（担当スタッフは残したまま指名追加）から方針変更。**担当スタッフ選択は撤去**し、指名の有無を選ぶボタンにする。実装前にこの改訂版を読み直すこと。

## 安全レベル
CAUTION（型変更＋予約台帳の作成/詳細/編集UIとドラッグ判定の変更。実行前にユーザー確認）

## 目的
予約の作成・詳細・編集画面から「担当スタッフ（必須）」の選択を**なくす**。担当は予約時点で確定させず、台帳上のドラッグで直前まで動かせる運用にする。
予約はタイムラインで**クリックした行のスタッフに自動で紐づく**ので、画面ではスタッフを選ばせない。代わりに**「指名する/しない」だけを切り替えるトグルボタン1つ**を置く（スタッフの再選択はさせない＝二度手間・入力ミス防止）。指名された予約はそのスタッフから別スタッフへ移動できないようにする。

## 背景
- 運用: 担当は予約状況により直前まで変わるため、予約入力時に担当を決める必要がない。
- お手本: 予約登録モーダルの**左下「スタッフを指名」ボタン**（ユーザー提供スクショ）。
- 対象の予約UI（作成/詳細/編集モーダル）とドラッグ判定は `features/reservations/reservation-ledger.tsx`。型は `features/reservations/types.ts`。

- `features/reservations/reservation-ledger.tsx`（担当スタッフ選択の撤去、指名ボタン、ドラッグ判定、表示）

## 実行内容
1. `Reservation` 型に `nominatedStaffId?: string`（指名スタッフ。未設定＝指名なし）を任意追加。
2. **予約作成/詳細/編集モーダルから「担当スタッフ」の選択UIと必須バリデーションを撤去**する。
   - `staffId` はフォームの内部状態としては保持する（画面では選ばせない）。
   - 空きセルクリックで開いた場合＝そのクリックしたスタッフ行の staffId を仮の担当にする。
   - 「新規予約」ボタン（セル無し）で開いた場合の既定担当＝**その日の先頭の出勤スタッフを仮置き**（後でドラッグ移動できる）。判断に迷えば停止して確認。
3. 「指名」トグルボタンを1つ追加（モーダル左下イメージ）。**スタッフ選択UIは作らない。**
   - 指名ON＝この予約の現在のスタッフ（＝クリックした行／ドラッグ後の現担当）を指名扱いにする。`nominatedStaffId = その予約の staffId` を保存。
   - 指名OFF＝`nominatedStaffId` を空にする（担当は自由にドラッグ移動可）。
   - ボタンは「指名」「指名なし」の2状態が一目で分かる見た目（押すたびにトグル）。
   - 注意: 指名スタッフを別途選ばせない。担当はあくまでタイムライン上のクリック/ドラッグで決まり、このボタンは「その担当を指名として固定するか否か」だけを切り替える。
4. ドラッグ判定（`validateDraggedReservation`）: `nominatedStaffId` 設定済みで移動先スタッフが異なる場合は移動不可（「指名予約のため担当スタッフは変更できません。」）。時刻移動はOK。
5. 予約カード/詳細に「指名」表示（指名ありが分かるバッジ等）。
6. スタッフ重複・シフト外・容量（個室/施術）の既存判定は維持。
