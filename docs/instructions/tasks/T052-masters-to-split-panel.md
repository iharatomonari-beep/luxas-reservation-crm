# タスクID：T052
# タスク名：各マスタ画面をPMのスプリットパネル形式に統一（1画面ずつ）

## 安全レベル
CAUTION（既存マスタ画面のUI差し替え。CRUDロジックは維持）

## 目的
T051の共通スプリットパネルを使い、各マスタをPM形式（左一覧＋右明細設定＋上部 名前検索/検索/検索条件追加/新規作成）に揃える。**1画面ずつ**実装・検証する。

## 対象（この順で1つずつ）
1. スタッフ（staff-manager）: 列 ID/名前/ニックネーム(定休日表記)/表示順。明細に既存項目。
2. ブース（room-manager）: 列 ID/名前/略称/表示順（種別/容量はLUXAS拡張として明細に残す）。
3. 経費マスタ（expense-accounts）: 列 ID/勘定科目/補助科目/表示順。
4. クレカ会社（creditcards）: 列 ID/会社名/手数料率%/表示順。
5. 電子マネー（emoney）: 列 ID/電子マネー/手数料率%/表示順。
6. 顧客シンプル（customers 一覧側）: 列 ID/会員番号/氏名/フリガナ（既存の充実検索は維持してよい）。
7. 商品カテゴリ（categories）: 列 ID/名前/表示順。

## 対象ファイル
- 各 features/master-data/*-manager.tsx, features/customers/customer-manager.tsx（該当の1ファイルずつ）
- components/master/master-split-panel.tsx（T051・参照）

## 実行内容（各画面共通）
- 既存のCRUD（追加/編集/削除/表示順）を維持したまま、レイアウトをスプリットパネルへ。
- 既存のLUXAS拡張フィールド（役割/容量/種別/対応メニュー等）は明細フォーム内に残す。
- 1画面実装→lint/build→実機確認→次へ。

## 変更してはいけないファイル
- localStorageキー・データ構造の破壊変更。import/export。

## 検証
- 各画面: npm run lint / npm run build 成功。実機でCRUDが従来どおり動く＋見た目がスプリットパネル。

## 完了条件
1. 対象7画面がスプリットパネル化（不足列も追加）。2. CRUD維持。3. lint/build OK。4. 各完了で_index更新＋sessions追記。5. 実機OKで完了。

## 停止条件
- データ構造変更が要るなら停止して報告。1画面でも壊れたらそこで停止。指定外ファイルはSTOP。

## 報告
- 各画面ごとに 変更ファイル / lint・build / 実機確認依頼。
