# タスクID：T015
# タスク名：シフト画面を「曜日別の出勤チェック＋時間設定」型にする（週次パターン→2ヶ月分生成）

## 安全レベル
CAUTION（シフト管理画面の作り直し。実行前にユーザー確認）

## 目的
スタッフごとに「出勤曜日のチェック」と「その曜日の勤務時間」を設定し、その週次パターンから実際のシフト（per-date）を生成できる画面にする。お手本は PeakManager の月間シフト＋Googleカレンダーの「毎週繰り返す」型（曜日ごとに開始〜終了、土日は出勤なしにできる）。

## 背景
- 現状のシフト管理（`features/master-data/shift-manager.tsx`）は1件ずつ日付指定で登録する形。これを曜日別の週次パターン入力に変える。
- シフトは per-date（`StaffShift.workDate`）。パターンから日付分を生成する（生成ロジックは T013 で用意するシフト生成関数を再利用する。無ければ本タスクで同等の関数を `mock-data.ts` に用意）。
- 休憩は空（`breakStart=""`/`breakEnd=""`）。

## 確定仕様（ユーザー確認済み）
- 生成期間: **今日から2ヶ月分**。
- 既存シフト: **対象期間内はそのスタッフ分を全上書き**（期間内を一度クリアしてパターンで再生成）。

## 対象ファイル
- `features/master-data/shift-manager.tsx`（曜日別パターン入力UIに作り直し）
- `features/master-data/mock-data.ts`（シフト生成関数の再利用／追加。型が必要なら最小限）
- 必要時のみ `features/master-data/types.ts`（週次パターン型を足す場合。迷えば停止）

## 変更してよいファイル

- `features/master-data/mock-data.ts`

## 実行内容
1. シフト管理画面を次のUIにする:
   - スタッフを選ぶ（または一覧でスタッフごとに）。
   - 月・火・水・木・金・土・日の各行に「出勤するか」のチェックボックスと、チェック時に有効な「開始時刻／終了時刻」入力。
   - 土日もチェックで出勤可（参考画像のように曜日単位で設定）。
2. 「2ヶ月分を作成」ボタン:
   - そのスタッフの **今日〜2ヶ月後** の既存シフトを一旦クリア（期間内全上書き）。
   - チェックされた曜日について、設定時刻で per-date のシフトを生成（`breakStart/End` 空、`isActive: true`、id 一意）。
3. 入力検証: 開始 < 終了、HH:mm 形式、チェック曜日は時刻必須。
4. 保存後、予約台帳に反映される（同じ localStorage キー `luxas-master-shifts-v2`）。
5. 週次パターンを再編集できるよう、パターンを保持するか既存シフトから復元するかは実装で判断し、理由を報告。
