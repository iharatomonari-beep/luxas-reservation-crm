# タスクID：T037
# タスク名：割引・インターバル・連続予約の計算/挙動を実装

## 安全レベル
CAUTION（予約作成・売上計算ロジック）

## 前提
T030（予約データ完全化＝割引/連続/オプション/インターバルの保存）が実装済みであること。未了なら先行として停止報告。

## 目的
T020/T030で保存した割引・インターバル・連続予約を、実際の計算/挙動に反映する。

## 背景
PM予約受付: 個別割引/一括割引(%・円)、インターバル(施術後の空け時間)、連続(続けて次枠を取る)。catalog §2。

## 対象ファイル / 変更してよいファイル
- `features/reservations/reservation-ledger.tsx`（作成モーダル・配置計算）
- `features/reservations/types.ts`（必要なら計算用フィールド）
- 参照: T024オプション（延長=時間/料金）

## 変更してはいけないファイル
- `features/customers/` `features/import-export/` `features/master-data/`
- `lib/supabase/` `supabase/` `.env*` `package.json` `tailwind.config.ts`、上記以外

## 実行内容
1. 割引: コース料金＋オプション料金から、個別割引(%→円換算)・一括割引を差し引いた売上額を算出し `saleAmount` に反映（会計T022と整合）。
2. インターバル: 施術時間＋インターバル分をカードの占有幅（終了時刻）に反映。重複/容量判定にも含める。既存のインターバルON/OFFと連動。
3. 連続予約: 「連続」ONで、同一顧客の次の枠を続けて作成する導線（最小=保存後に続けて次の開始時刻をプリセットして再度モーダル）。
4. 計算の端数処理（四捨五入等）は推奨で決めて報告。

## 検証方法
- `npm run lint` / `npm run build`／実機でユーザー確認。

## 完了条件
1. 割引が売上額に反映、インターバルが幅/判定に反映、連続予約の導線が動く。2. 既存を壊さない。3. lint/build OK。4. sessions追記・_index更新。5. 実機OKで完了。

## 停止条件
- T030未了なら停止。端数処理・連続予約UXは推奨で進め報告。容量判定への影響が読めない場合は確認。指定外ファイルはSTOP。lint/build失敗で停止。

## 完了後の報告内容
- 完了ID/割引計算式/インターバル反映/連続予約UX/lint・build/_index更新/実機確認依頼
