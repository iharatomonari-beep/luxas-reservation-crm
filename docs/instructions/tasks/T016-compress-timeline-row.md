# タスクID：T016
# タスク名：予約台帳タイムラインの行高さを約2/3に圧縮し、予約カードのテキストを枠内に収める

## 安全レベル
CAUTION（予約台帳のUI変更。実行前にユーザー確認）

## 目的
スタッフが縦に多く並んでも全員見えるよう、タイムラインの1行（スタッフ行）の高さを約2/3に圧縮する。あわせて予約カード内のテキストの縦間隔を調整し、短くなったカードからはみ出さず収まるようにする。

## 背景
- 現状、1行の高さは `timelineRowHeight = 96`（`features/reservations/reservation-ledger.tsx`）。スタッフが増えると縦に長く全員表示できない。
- 予約カードには顧客名・メニュー・時刻・ブース種別などが縦に並ぶため、行高さだけ縮めるとテキストがはみ出る。行間・余白・フォントも合わせて詰める必要がある。

## 対象ファイル
- `features/reservations/reservation-ledger.tsx`

## 変更してよいファイル
- `features/reservations/reservation-ledger.tsx` のみ

## 変更してはいけないファイル
- `features/customers/`、`features/import-export/`、`features/master-data/`
- `lib/supabase/`、`supabase/`、`app/`、`.env` / `.env.local`、`package.json`、`tailwind.config.ts`
- 上記以外すべて

## 実行内容
1. `timelineRowHeight` を現状(96)の **約2/3（目安64）** に下げる。スタッフ列の行高さ・予約カードの高さ計算もこれに追従させる（行高さに依存している箇所をすべて更新）。
2. 予約カード内テキストの縦調整:
   - 行間（line-height）・上下padding・要素間マージンを詰める。
   - フォントサイズを必要に応じて1段階小さく。
   - 収まらない場合は省略（truncate）や表示項目の優先順位付けで、**カード枠からはみ出さない**ようにする。
3. 5分グリッド・時刻ヘッダー・ドラッグ・クリック・色分け（シフト内外）は壊さない。
4. 圧縮率は目安2/3。テキストが収まらない/詰まりすぎる場合は微調整し、判断に迷う見た目は「要確認」として報告。
