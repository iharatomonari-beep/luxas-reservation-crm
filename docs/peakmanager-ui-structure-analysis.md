# PeakManager UI Structure Analysis

この文書は、現時点で作業ツリーにある PeakManager の次回エクスポート確認メモ、サンプルCSV、画面一覧をもとに、業務構造だけを整理したものです。UIの色、CSS、余白、アイコン、文言、ロゴは LUXAS に持ち込みません。

## 1. PeakManager の上位メニュー構造

PeakManager の画面群は、日々の受付業務とマスタ管理と経営確認に分かれています。

- 予約台帳
- 日次管理
- 店舗情報
- 顧客情報
- メール管理
- 商品情報
- 経営指標

補助的に見るべきものとして、店舗マスタ、スタッフマスタ、ブースマスタ、売上/来店状況のような管理系画面があります。

## 2. 予約台帳系の画面構成

予約台帳系は、受付現場が当日の空きと重複を即座に判断するための中心画面です。

- 日別の予約一覧
- 時刻順の並び
- 担当スタッフ別の把握
- ブース別の衝突確認
- 予約詳細
- 予約作成
- 予約編集
- 予約キャンセル
- 予約検索

LUXAS が取り込むべき構造は、日付を軸にした予約一覧、担当スタッフ別表示、空き枠から作成へ進む導線、詳細から編集やキャンセルへ進む導線です。

## 3. 日次管理系の画面構成

日次管理系は、当日の営業状況を横断して確認するための領域です。

- 当日の予約
- 来店状況
- 売上の確認
- キャンセルの確認
- スタッフ稼働の確認
- ブース稼働の確認

LUXAS では、日次管理を予約台帳の補助ではなく、予約台帳と顧客詳細をつなぐ中間層として扱うのがよいです。将来的には来店実績、売上、キャンセルをまとめて見られる日報画面にできます。

## 4. 店舗情報系の画面構成

店舗情報系は、店舗運営の前提条件をまとめる領域です。

- 店舗設定
- 営業時間
- 予約受付時間
- 予約単位
- 表示単位
- スタッフ在籍
- ブース稼働

LUXAS では、店舗設定を単なる表示名ではなく、予約可否判定とタイムラインの基準値として扱います。

## 5. 顧客情報系の画面構成

顧客情報系は、検索と詳細参照とカルテの3層で整理できます。

- 顧客検索
- 顧客詳細
- 注意事項
- 予約履歴
- カルテメモ
- 来店実績
- 会員情報

LUXAS に必要なのは、顧客検索の即応性、詳細画面での注意事項と予約履歴、カルテ追記のしやすさです。PeakManager の表示順や装飾はコピーせず、検索と参照の順序だけを参考にします。

## 6. メール管理系の画面構成

メール管理系は、現時点の LUXAS v0.1 では中心機能ではありません。

- 予約案内
- リマインド
- 来店後フォロー
- 一斉配信

LUXAS では、v0.1 ではこの領域を実装対象外とし、顧客や予約の基本構造が固まってから検討します。

## 7. 商品情報系の画面構成

商品情報系は、メニューや物販を管理する領域です。

- メニュー
- 所要時間
- 価格
- 有効/無効
- 表示順
- 関連スタッフ
- 関連ブース

LUXAS では、予約作成に必要なメニューマスタだけを先に持ち、物販や販売施策は後回しにします。

## 8. 経営指標系の画面構成

経営指標系は、日次管理よりさらに集約した分析領域です。

- 売上
- 来店数
- 取消数
- 無断キャンセル数
- 客単価
- 稼働率
- スタッフ別集計
- メニュー別集計

LUXAS の v0.1 では、ここは構造だけを把握しておき、画面実装は後に回します。

## 9. LUXAS に取り込むべき構造

- 予約台帳を中心にする構造
- 日付で絞り込む構造
- スタッフごとの行または列で見る構造
- 空き枠から作成に進む構造
- 顧客詳細で予約履歴とカルテをつなぐ構造
- スタッフ、ブース、メニューを別マスタに分ける構造
- 予約と来店や売上を分けて考える構造

## 10. LUXAS に取り込まない構造

- PeakManager の配色
- PeakManager の余白やカードの見せ方
- PeakManager の文言
- PeakManager のロゴやアイコン
- 画面遷移の名前のままの再現
- 店舗切り替え前提の構造
- メール管理を v0.1 の中心に置く構造

## 11. 参考にする方針

PeakManager は、UI の見た目ではなく、業務の分解方法だけを参考にします。

- 受付が何を見ているか
- 予約作成時に何を固定したいか
- マスタがどこで必要か
- 顧客詳細に何を集約するか
- 日報や経営指標に何を送るか

## 12. 今後の実装順

- 予約台帳の表示条件と予約作成導線
- 店舗設定の営業時間と予約単位
- スタッフ、ブース、メニューの最小マスタ
- 顧客詳細の予約履歴と注意事項

P1

- シフトパターン
- メニュー別利用可能ブース
- スタッフ別対応メニュー
- 予約ルートタグ
- 施術カルテタグ

P2

- 日次管理の集計
- 売上/来店状況
- メール管理
- 経営指標
