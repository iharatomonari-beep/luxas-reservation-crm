# テストチェックリスト

## 画面表示

- `/login` が表示できる。
- `/dashboard` が表示できる。
- `/dashboard/staff` が表示できる。
- `/dashboard/services` が表示できる。
- `/dashboard/rooms` が表示できる。
- `/dashboard/shifts` が表示できる。
- `/dashboard/reservations` が表示できる。
- `/dashboard/customers` が表示できる。
- `/dashboard/import-export` が表示できる。

## 認証

- 未ログインユーザーが業務画面へアクセスできない。
- ログイン済みスタッフが予約台帳へ遷移できる。
- ログアウト後に保護画面へ戻れない。

## 予約台帳

- 日付を切り替えると対象日の予約だけが表示される。
- 予約の新規作成、編集、キャンセルができる。
- `booked`, `arrived`, `completed`, `cancelled`, `no_show` の表示が崩れない。
- 顧客、スタッフ、ブース、メニューが予約に表示される。
- 開始時刻より終了時刻が前の予約を保存できない。

## スタッフ別時間グリッド

- 営業時間と予約単位に合わせて時間軸が表示される。
- スタッフごとに予約が正しい位置に表示される。
- シフト外の時間が判別できる。
- 同一スタッフの重複予約で警告が出る。
- グリッドから予約作成または編集へ進める。

## 顧客検索・カルテ

- 氏名、フリガナ、電話番号、メール、タグで検索できる。
- 顧客の新規作成と編集ができる。
- 顧客詳細で予約履歴が確認できる。
- カルテメモを追加できる。
- 注意事項が予約作成時または顧客詳細で確認できる。

## CSV入出力

- 顧客CSVをプレビューして取り込める。
- スタッフCSVをプレビューして取り込める。
- メニューCSVをプレビューして取り込める。
- 予約CSVをプレビューして取り込める。
- 顧客CSVを書き出せる。
- スタッフCSVを書き出せる。
- メニューCSVを書き出せる。
- 予約CSVを書き出せる。
- 必須項目がない行は取り込まれない。
- 成功件数と失敗件数が表示される。

## シフト・ブース・メニュー

- スタッフの有効・無効状態が予約画面に反映される。
- シフト登録、編集、削除相当の操作ができる。
- ブースの有効・無効状態が予約画面に反映される。
- 同一ブースの重複予約で警告が出る。
- メニューの所要時間が予約終了時刻の初期値に反映される。

## データ・権限

- ログイン済みスタッフだけが業務データを読める。
- SupabaseのRLSで匿名アクセスが拒否される。
- 必須項目が空のまま保存できない。
- エラー時に画面が空白にならず、再試行できる。

## UI確認

- デスクトップ幅で予約台帳と時間グリッドが読みやすい。
- 最小限のモバイル幅で主要情報が重ならない。
- ローディング、空状態、エラー状態が表示される。
- PeakManagerのUI、CSS、画像、アイコン、文言をコピーしていない。

## デプロイ確認

- Vercel環境変数が設定されている。
- 本番URLでログインできる。
- AuthリダイレクトURLが本番URLと一致している。
- 主要フローを本番URLで一通り実行できる。
