## 3. PM準拠で再現済みのUI（参考スクショと一致）
- 予約台帳タイムライン：基本/全体、5分刻み、シフト内外グレー、ドラッグ移動、**現在時刻の赤線**、**時間ジャンプバー＋件数**、**前後インターバルのグレー枠**

## 5. 重要な前提・既知の注意（Coworkも把握しておくと良い）
- **本部アカウントと店舗アカウントでメニュー構成が違う**（PM実機で確認）。
  - 本部：予約台帳 / 本部 / 顧客情報 / メール管理 / **商品情報[本部]** / 経営指標 / **アプリ管理** / ヘルプ
  - 店舗：予約台帳 / 日次管理 / 店舗情報 / 顧客情報 / 商品情報 / メール管理 / 経営指標
  - → 「全店共通コース＝本部が色・内容を管理」「店舗独自＝店舗が管理」の前提が裏取りできた。
- データは**オリジン別 localStorage**。Vercel（本番）と localhost は別データ。コードが同じでも中身は揃わない（仕様）。
- 開発運用ルール（崩さない）：dev稼働中は `npm run build` しない（`.next`破損→500の原因）。型チェックは `npx tsc --noEmit`。UI変更は1つずつ Chrome 確認＋コンソールエラー確認→小刻みに commit/push。
