## 2. 未実装・拡充候補（優先度つき）

### A. 会計まわり（優先度：高）
1. **物販購入の本実装**（会計画面の「物販購入」が現在は準備中）
   - 商品選択（物販マスタから）／商品入力（手入力）／個別値引・一括値引／物販合計
   - 物販売上を総販売額・集計バー（物販 ¥）に反映
2. **回数券・プリペイドの本実装**（左レール「カード」ボタンが準備中）
   - 回数券：販売・利用・残回数管理
   - プリペイド：チャージ（販売）・利用・残高管理
   - ポイント：付与・利用・残高
   - 商品券：利用
   - 会計画面の「回数券販売」「プリペイド販売」「ポイント付与」を実値化
3. **コース個別値引・一括値引**（会計画面の「コース個別値引一括設定」%ボタン）
4. **EPARKサービス**金額の扱い（現在は入力欄のみ）

### B. 予約まわり（優先度：中）
5. **複数コース予約**（PMは1予約に複数コース＋物販をぶら下げ可能）。現状は1予約=1コース。
6. **セット商品（CourseSet）を予約・会計に反映**（マスタは存在＝T053、予約導線は未接続）
7. **gender未設定の既存顧客への補完**（紐づけ時に任意で Customer.gender を補完）
8. **施術コメントの顧客カルテ蓄積**（現状は予約単位 memo。顧客 chartMemo への蓄積は要相談）

### C. マスタ・カタログ（優先度：中）
9. **実コースカタログの拡充**（マッサージ30/45/60/75/90/105/120…、出張、鍼、キッズ、マタニティ等）
   - 現状はサンプル数件。PMのような duration バリエーション一覧が必要（マスタ追加で対応可・seed大改変は避ける）
10. **コース色の店舗別保存**（現状は1コース1色。店舗ごとに同一コースの色を変えたい場合は色フィールドの店舗別化が必要）
11. **オプションのカテゴリ整理**（男性希望/女性希望/延長/割引系など。割引系は会計の値引と連動させる設計）

### D. 顧客・CRM（優先度：中〜低）
12. **顧客カルテ画面の充実**（来店履歴・施術コメント履歴・売上推移・写真等）
13. **返客の集計・分析**（時間帯別・理由別の返客数→機会損失の可視化。日次/経営指標に反映）
14. **新規顧客作成導線**（返客/予約の検索パネルの「＋新規」が準備中）

### E. タイムライン/運用（優先度：低〜中）
15. **ブース（ベッド）ビュー／シフト＋ブースの両方ビュー**（ボタンは準備中）
16. **保留棚→別日配置のUX改善**（既存あり、磨き込み）
17. **天気・絞り込みなどヘッダ補助の実値化**（現在プレースホルダ）

### F. その他（優先度：低）
18. **操作説明/ヘルプ**ページ
19. **権限・ロール**（本部/店舗/スタッフ）— 色設定の「本部/店舗」など現状は注記のみ。認証導入後。
20. **Supabase 移行**（現状 localStorage。v0.2 以降）

## 4. 既知の運用ルール（崩さない）
- dev サーバ稼働中は `npm run build` を実行しない（`.next` 破損→500の原因）。型は `npx tsc --noEmit`。
- UI変更は毎回 Chrome 新規タブ＋コンソールエラー確認 → 小刻みに commit/push。
- localStorageキー・seed の大改変はしない（非破壊追加で対応）。
