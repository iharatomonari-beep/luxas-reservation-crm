# PeakManager 次回エクスポート確認チェックリスト

この文書は、PeakManager 管理画面から次に取得できる出力ファイルを確認するためのチェックリストです。

前提:
- 本物の顧客情報はまだ取り込みません。
- `localStorage` は検証用です。
- Supabase 本接続前に本番データは入れません。
- 取得できたファイルは、まずヘッダーだけ確認します。
- その後、ダミーCSVを作って取り込み機能を実装します。

## 1. 次に PeakManager 管理画面で探すべき出力ファイル一覧

優先順:

1. 予約履歴CSV
2. 来店履歴CSV
3. 売上Excel
4. スタッフCSV
5. メニューCSV
6. 予約一覧CSV
7. 顧客一覧CSV
8. キャンセル履歴CSV
9. 会計明細CSV

## 2. 各ファイルの目的

### 予約履歴CSV

- 予約の作成・変更・キャンセルの流れを確認するための基礎データです。
- LUXAS の予約台帳に近い項目を持っている可能性があります。

### 来店履歴CSV

- 実際に来店した履歴を確認するためのデータです。
- 顧客詳細の「初回来店」「最終来店」「総来店回数」の補完に使える可能性があります。

### 売上Excel

- 顧客単位、予約単位、日別、月別などの売上確認に使う可能性があります。
- LUXAS の売上管理を将来追加する場合の参考になります。

### スタッフCSV

- スタッフ名、表示名、担当可否、在籍状態などの確認に使います。
- LUXAS のスタッフ管理に対応させやすいです。

### メニューCSV

- メニュー名、所要時間、価格、表示順などの確認に使います。
- LUXAS のメニュー管理に対応させやすいです。

### 予約一覧CSV

- 予約履歴CSVと似ていても、列構成や粒度が違う場合があります。
- 予約台帳取り込みの候補になります。

### 顧客一覧CSV

- 顧客マスタの基本項目を確認できます。
- 顧客明細CSVより軽量な場合は、最初の取り込み候補になります。

### キャンセル履歴CSV

- キャンセル理由、キャンセル日時、無断キャンセルの扱いを確認するために使います。
- 顧客詳細の取消回数や無断キャンセル数の補完に使える可能性があります。

### 会計明細CSV

- 予約や顧客に紐づく売上の粒度を確認するために使います。
- 売上Excelより細かい場合があります。

## 5. 予約履歴CSVが取れた場合のマッピング方針

候補:
- 顧客名 -> `customerName`
- 電話番号 -> `customerPhone`
- 予約日 -> `date`
- 開始時刻 -> `startTime`
- 終了時刻 -> `endTime`
- メニュー -> `serviceName`
- 担当スタッフ -> `staffName`
- ブース -> `roomName`
- ステータス -> `status`

方針:
- まずはヘッダーを確認し、LUXAS の予約データ型に合わせて列名を決めます。
- 顧客名と電話番号があれば、顧客既存データとの紐づけ候補にします。
- 同一スタッフ・同一ブースの重複チェックに使えるかを確認します。

## 6. 来店履歴CSVが取れた場合のマッピング方針

候補:
- 顧客名 -> `name`
- 会員番号 -> `membershipNumber`
- 来店日 -> `lastVisitDate` または `firstVisitDate` の補完材料
- 来店回数 -> `totalVisits`
- 担当スタッフ -> `lastVisitStaff`
- 店舗 -> `lastVisitStore`
- 売上 -> `totalSalesIncTax`
- キャンセル回数 -> `cancelCount`
- 無断キャンセル -> `noShowCount`

方針:
- 来店履歴は、顧客詳細の履歴補完に使います。
- 顧客単位で集約できるなら、最後に顧客データへ反映します。
- 予約履歴と別管理の場合は、履歴テーブルとして保持する案も検討します。

## 7. 売上Excelが取れた場合のマッピング方針

候補:
- 日付 -> `saleDate`
- 顧客名 -> `customerName`
- メニュー -> `serviceName`
- 担当スタッフ -> `staffName`
- 売上金額 -> `amount`
- 税区分 -> `taxType`
- 支払方法 -> `paymentMethod`
- 値引き -> `discount`

方針:
- まずはヘッダーを確認して、Excel 由来の列を CSV に変換できるかを見ます。
- LUXAS では、売上は集計保存か、予約単位に紐づく補助情報として持つかを決めます。
- 顧客詳細の総売上金額（税込）に反映できるかを確認します。

## 8. スタッフCSV・メニューCSVが取れた場合のマッピング方針

### スタッフCSV

候補:
- スタッフ名 -> `name`
- 表示名 -> `displayName`
- 在籍状態 -> `isActive`
- 役割 -> `role`

方針:
- 既存のスタッフ管理の型に合わせて、足りない項目は追加候補にします。
- 退職者アカウント停止の運用にもつなげます。

### メニューCSV

候補:
- メニュー名 -> `name`
- 所要時間 -> `durationMinutes`
- 価格 -> `price`
- 表示順 -> `sortOrder`
- 有効/無効 -> `isActive`

方針:
- 予約作成フォームの選択肢にそのまま使える構造を優先します。
- 価格や所要時間が取れるなら、予約作成の補完に使います。

## 9. 本物データを入れる前の安全ルール

1. まずヘッダーだけ確認します。
2. 個人情報が入っている行は、業務責任者の確認が取れるまで開かない前提にします。
3. 取り込み確認はダミーCSVで先に行います。
4. 本物のCSVは、個人情報を削除したものだけを使います。
5. 本物の顧客情報を localStorage に入れません。
6. Supabase 本接続前に本番データを使いません。
7. 取り込み前プレビューで、対象件数と列名を確認します。
8. 不明な列は捨てず、まず保留にします。

## 10. 個人情報を消したダミーCSVで検証する手順

1. PeakManager 管理画面から出力ファイルのヘッダーを確認します。
2. 実データは削除し、ダミー値だけの CSV を 3 件から 5 件作ります。
3. 1行目のタイトル行、2行目のヘッダー行、3行目以降のデータ行を揃えます。
4. LUXAS 側の取り込み前プレビューで列対応を確認します。
5. 必須項目が通るか確認します。
6. 取り込み後に一覧画面と詳細画面で表示を確認します。
7. 履歴や数値項目が正しく保存されるか確認します。
8. 問題がなければ、実データ版に進む前の手順書を更新します。

## 12. 取り回しメモ

- 取得できたファイルは、まずヘッダーだけ共有してください。
- CSV か Excel かで、取り込み準備の順序が変わります。
- 予約履歴と来店履歴が両方ある場合は、どちらを正にするか先に決めます。
- 売上Excelは、最初は列の確認だけで十分です。
