# 1週間プロトタイプ計画

## Day 1: 基盤と情報設計

- Next.js, TypeScript, Tailwind CSS の初期構成を作る。
- Supabaseプロジェクト、Auth、環境変数、Vercel連携方針を確認する。
- DBスキーマ、画面一覧、主要導線を確定する。
- 仮データの種類を決める。

## Day 2: 認証とアプリシェル

- スタッフログイン、ログアウト、保護ルートを作る。
- 共通レイアウト、ナビゲーション、日付切り替えの土台を作る。
- Supabaseクライアントと型の扱いを整える。
- スタッフ、メニュー、ブースの初期データを入れる。

## Day 3: 予約台帳

- 日別予約一覧と予約ステータス表示を作る。
- 予約作成、編集、キャンセルを実装する。
- 顧客、スタッフ、ブース、メニューを予約に紐付ける。
- 時間重複の基本チェックを入れる。

## Day 4: スタッフ別時間グリッド

- 営業時間と予約単位に基づく時間グリッドを作る。
- スタッフ別に予約、空き、シフト外を表示する。
- グリッドから予約作成・編集へ遷移できるようにする。
- 表示密度と読みやすさを調整する。

## Day 5: 顧客検索とカルテ

- 顧客検索、顧客作成、顧客詳細、カルテメモを作る。
- 来店履歴、予約履歴、注意事項、タグを表示する。
- 顧客データを localStorage に保存する。
- 予約データから電話番号または顧客名で関連予約を表示する。

## Day 6: CSV入出力

- CSVインポート／エクスポート画面を作る。
- 顧客、スタッフ、メニュー、予約のCSV入出力を実装する。
- インポート前プレビューとエラー表示を整える。
- サンプルCSVを用意して、localStorageデータとつなげる。

## Day 7: 検証、調整、デプロイ

- テストチェックリストに沿って主要フローを確認する。
- モバイル最小対応とデスクトップ業務画面の崩れを直す。
- Vercelにデプロイし、環境変数とAuthリダイレクトを確認する。
- 残課題、v0.2候補、既知の制限を記録する。
