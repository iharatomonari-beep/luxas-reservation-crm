# LUXAS予約・顧客管理システム v0.1 プロトタイプスコープ

## 目的

1週間で、単一店舗の受付・施術現場が予約状況、顧客情報、スタッフ稼働、ブース利用を確認・更新できる内部向けプロトタイプを作る。完成度よりも、主要業務フローが画面上で検証できることを優先する。

## 技術前提

- フロントエンド: Next.js, TypeScript, Tailwind CSS
- バックエンド: Supabase PostgreSQL, Supabase Auth
- デプロイ: Vercel
- 店舗前提: single store。将来拡張を妨げないため、主要テーブルには必要に応じて `store_id` を持たせる。

## v0.1で作る範囲

- Supabase Auth によるスタッフログイン
- 日別の予約台帳
- スタッフ別時間グリッド
- 予約の作成、編集、キャンセル、ステータス変更
- 顧客検索、顧客詳細、簡易カルテ
- スタッフ一覧とシフト登録
- ブース一覧と予約へのブース割り当て
- 施術メニューの基本管理
- 単一店舗の営業時間、予約単位、表示設定

## v0.1で作らない範囲

- オンライン予約
- LINE連携
- 決済
- ポイント
- 回数券
- メール配信
- 売上分析
- 複数店舗管理
- 高度な権限管理
- 自動リマインド、在庫、会計連携

## UI/UX制約

PeakManagerの業務要件は参考にしてよいが、UI、CSS、画像、アイコン、文言はコピーしない。LUXAS向けに、静かで業務的な画面密度、明確な状態表示、素早い検索と編集を重視する。

## 成功条件

- 受付スタッフが当日の予約状況を3秒以内に把握できる。
- 予約の新規作成から顧客カルテ更新までの流れをデモできる。
- スタッフ別の空き時間とブース利用が同じ業務文脈で確認できる。
- Vercel上のプロトタイプURLで、ログイン後の主要画面を確認できる。
