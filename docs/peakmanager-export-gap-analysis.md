# PeakManager Export Gap Analysis

この文書は、現時点で確認できる PeakManager 関連の出力サンプルと移行チェックリストをもとに、LUXAS の再現に足りない情報を整理したものです。

## 1. 既に入手したCSV/XLSの一覧

現時点で作業ツリーで直接確認できるものは次のとおりです。

- `docs/sample-csv/peakmanager-customers-sample.csv`
- `docs/sample-csv/customers.csv`
- `docs/sample-csv/staff.csv`
- `docs/sample-csv/services.csv`
- `docs/sample-csv/reservations.csv`

PeakManager 由来として直接参照できるのは、顧客明細のサンプルCSVです。その他の出力候補は `docs/peakmanager-next-export-checklist.md` に整理されていますが、実ファイルはまだ収集途中です。

## 2. 各ファイルで分かること

### `peakmanager-customers-sample.csv`

- 顧客ID
- 顧客名
- カナ
- 性別
- 電話番号
- 会員番号
- メールアドレス
- ランク
- 顧客タグ
- コメント
- 初回来店日時
- 初回来店店舗
- 最終来店日時
- 最終来店店舗
- 総来店回数
- 総売上金額
- 取消
- 無断キャンセル

### `customers.csv`

- LUXAS の顧客マスタの基本項目
- 検索に必要な最小項目
- 注意事項やカルテの保存先候補

### `staff.csv`

- スタッフ名
- 表示名
- 役割
- 在籍状態
- 並び順

### `services.csv`

- メニュー名
- 所要時間
- 価格
- 有効/無効
- 表示順

### `reservations.csv`

- 予約の基本構造
- 日付
- 開始時刻
- 終了時刻
- 担当
- ブース
- 状態
- メモ

## 3. 各ファイルで分からないこと

### 顧客明細CSVで分からないこと

- 予約時の入力順
- 予約台帳での配置ルール
- 日次の表示対象スタッフ
- シフトとの整合
- ブース利用ルール
- 予約受付締切

### 顧客・スタッフ・メニュー・予約のCSVで分からないこと

- 店舗の営業時間
- 予約開始と締切の境界
- 5分単位などの表示単位
- シフトパターン
- スタッフ別対応メニュー
- メニュー別利用可能ブース
- 予約ルート情報
- カルテタグ

## 4. 予約台帳再現に不足している情報

- 店舗設定
- 営業時間
- 予約受付時間
- 予約締切時間
- 時間表示単位
- 当日の有効シフト
- スタッフ別対応メニュー
- メニュー別利用可能ブース
- 予約ルート
- 予約作成時の固定条件

## 5. 追加で PeakManager から取得すべきデータ

### 優先度が高いもの

- 予約履歴CSV
- 来店履歴CSV
- スタッフCSV
- メニューCSV
- 予約一覧CSV
- キャンセル履歴CSV

### あると設計が安定するもの

- 店舗設定出力
- ブース設定出力
- シフト出力
- 売上Excel
- 会計明細CSV
- 顧客一覧CSV

## 6. CSV/XLS だけでできること

- 顧客マスタの初期投入
- スタッフマスタの初期投入
- メニューマスタの初期投入
- 予約の初期投入
- 顧客の PeakManager 情報表示
- 顧客検索の補強

## 7. API または手動マスタ登録が必要なこと

- 店舗営業時間
- 予約受付締切
- 5分単位の運用ルール
- シフトパターン
- 当日シフトの有効化
- スタッフ別対応メニュー
- メニュー別利用可能ブース
- 予約ルートタグの運用
- 施術カルテタグの運用

## 8. LUXAS 側の設計に対する示唆

- CSV は取り込みの入口として使う
- 営業ルールは別の店舗設定として持つ
- 予約可否はシフトと営業時間で判定する
- 顧客明細は検索と詳細の補助情報として扱う
- 経営指標は予約や顧客とは別の集計層として分離する

## 9. 実装順

P0

- 顧客、スタッフ、メニュー、予約の基本取り込み
- 顧客明細CSVの表示
- 予約台帳に必要な予約データ型の固定

P1

- 予約履歴CSVと来店履歴CSVの追加取得
- シフトと店舗設定の分離
- ブースとメニューの対応ルール

P2

- 売上Excel
- 会計明細CSV
- 経営指標に必要な追加出力
