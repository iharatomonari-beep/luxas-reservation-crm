# 手動確認メモ

## 確認環境

- ローカル開発環境
- ブラウザ: 任意の最新ブラウザ
- 状態保存: localStorage

## 2026-05-15 予約作成保存反映の確認

- `/dashboard/reservations/new` の保存先と `/dashboard/reservations` の読み込み元が同じ `luxas-reservations-sample` であること。
- 保存時の `date` が `YYYY-MM-DD`、`startTime` / `endTime` が `HH:mm` で保存されること。
- `/dashboard/reservations?date=保存日` の `date` と保存済み予約の `date` が一致し、該当日の予約数に含まれること。
- 保存時の `staffId` が台帳側の `staff.id` と一致し、該当スタッフ行に横長カードが表示されること。
- 保存した `startTime` / `endTime` の時間帯にカードが表示され、カード内の時間が `10:00 - 11:00` のように横一列で読めること。
- 不正な日付、時刻、営業時間外のデータで `NaN` やマイナス位置のカードが出ないこと。
- 保存後に予約台帳へ戻ると、該当日の localStorage データが再読み込みされること。
- 予約カードのクリック、詳細、編集、キャンセルが引き続き動くこと。
- この確認ではドラッグ移動、スタッフまたぎ移動、リサイズは対象外とすること。

## 2026-05-15 予約作成タブ自動反映の確認

- 予約台帳と予約作成ページを同じ origin（例: どちらも `http://localhost:3000`）で開くこと。`3000` と `3001` を混ぜると localStorage が分かれるため反映確認に使わないこと。
- `/dashboard/reservations` の空き枠クリックで、`staffId`、`date`、`startTime` 付きの `/dashboard/reservations/new` が新しいタブで開くこと。
- 新規予約タブで保存後、`window.opener.postMessage` または localStorage 更新通知で元の予約台帳タブが予約データを再読み込みすること。
- 保存後に新規予約タブが自動で閉じ、元の予約台帳タブに戻ること。
- 自動で閉じられない場合も、保存完了表示と「予約台帳へ戻る」リンクが表示されること。
- 元の予約台帳タブが保存日の `/dashboard/reservations?date=YYYY-MM-DD` 表示にそろい、保存した予約が該当スタッフ行・該当時間帯に横長カードで表示されること。
- 予約台帳タブを focus したときにも localStorage の `luxas-reservations-sample` が再読み込みされること。
- 予約カードの時間表示が `10:00 - 11:00` の横一列を維持すること。

## 2026-05-16 予約台帳からの作成導線確認

- `/dashboard/reservations?date=2026-05-11` を開いたとき、当日の有効シフトがあるスタッフだけがタイムラインに表示されること。
- 空き枠クリックで `staffId`、`date`、`startTime` を含む `/dashboard/reservations/new?staffId=...&date=...&startTime=...` が開くこと。
- 作成画面では、タイムライン由来の `staffId`、`date`、`startTime` が初期入力されること。
- タイムライン由来の予約作成では、日付・担当スタッフ・開始時刻が固定表示になること。
- 保存時の `date` が `YYYY-MM-DD`、`startTime` / `endTime` が `HH:mm` で保存されること。
- 保存後、新規予約タブが閉じるか、閉じない場合は保存完了表示と戻るリンクが出ること。
- 元の予約台帳タブで `reservation-created` の通知後に再読み込みされ、保存した予約が該当スタッフ行・該当時間帯に表示されること。
- 画面上の debug 表示で、作成画面は received 値と save payload、台帳は selectedDate と件数が見えること。

## 確認順

1. `/login` を開く。
2. `/dashboard` を開く。
3. `/dashboard/staff`、`/dashboard/services`、`/dashboard/rooms`、`/dashboard/shifts` を開く。
4. `/dashboard/reservations` を開いて予約を作成、編集、キャンセルする。
5. `/dashboard/shifts` を開いてシフトを作成、編集、削除する。
6. `/dashboard/customers` を開いて顧客を作成、検索、詳細確認、カルテ保存をする。
7. `/dashboard/import-export` を開いて CSV をプレビュー、取り込み、書き出しする。
8. PeakManager顧客明細CSVを取り込んで、顧客詳細の PeakManager情報を確認する。
9. `docs/sample-csv/peakmanager-customers-sample.csv` で表示イメージを確認してから取り込み手順を試す。
10. 予約台帳の空き枠をクリックして、`/dashboard/reservations/new` が別タブでスタッフ・日付・開始時刻付きで開くことを確認する。
11. 予約台帳の空き枠が 5 分単位で表示され、`10:05`、`10:10`、`10:15` のような開始時刻が渡ることを確認する。
12. `/dashboard/reservations/new` で保存した予約が、`/dashboard/reservations?date=YYYY-MM-DD` に戻ったとき横向きタイムラインへ反映されることを確認する。
13. 予約カードを同じスタッフ行内でドラッグし、5 分単位で時間がずれることを確認する。
14. 既存予約と重なる場所へドラッグすると、移動が止まり元の位置へ戻ることを確認する。

## 重点確認

- 予約台帳の右上から別タブで新規予約ページを開きやすいこと。
- 予約台帳が横軸を時間、縦軸をスタッフにした横向きタイムラインで表示されること。
- 10:00〜23:00 の時間ヘッダーと5分単位のグリッドが横スクロールで確認できること。
- 1時間ごとの見出しが読みやすく、15分ごとの線は控えめ、5分ごとの線はさらに薄く見えること。
- 保存済み予約が、表示日・担当スタッフ・開始終了時刻に合わせて該当スタッフ行の中に横長カードで表示されること。
- 予約カードがスタッフ行の外や画面下部にずれて表示されないこと。
- 保存した予約が別タブからでも台帳へ反映されること。
- 空き枠クリックで別タブの予約作成ページを開けること。
- 空き枠セル全体と「＋」の両方から別タブの予約作成ページを開けること。
- 新規予約ボタンと空き枠クリックの両方で、予約作成ページが確実に開くこと。
- クリックしたスタッフ、日付、開始時刻が予約作成ページに入ること。
- 予約カードに顧客名、メニュー、時間、ブース、注意事項の有無が見えること。
- 予約カードの時間が `10:00 - 11:00` のように横一列で見えること。
- 選択中の予約が上部の帯でも確認できること。
- 予約詳細から編集、キャンセル、顧客管理確認に進めること。
- 予約作成フォームが顧客情報から順に入力しやすいこと。
- メニュー選択後に終了時刻が補完されること。
- 顧客が登録済みなら注意事項が表示されること。
- 保存前要約で内容を確認できること。
- トップページから予約台帳、顧客管理、CSV入出力にすぐ入れること。
- 予約台帳で選択日の内訳とスタッフ別件数が見えること。
- 予約の重複警告が出ること。
- 顧客の予約履歴が表示されること。
- 顧客詳細の上部で注意事項が目立つこと。
- 顧客詳細の PeakManager情報で会員番号、ランク、総来店回数、総売上金額（税込）、取消、無断キャンセル、初回来店店舗、最終来店店舗が見えること。
- カルテメモが予約履歴より上にあること。
- 顧客詳細から予約作成へ進めること。
- シフト管理で勤務時間、休憩時間、状態、メモが見えること。
- 予約作成時にシフト外、休憩時間、対応外メニュー、停止中ブースで分かりやすく止まること。
- 同じスタッフ行内の予約カードをドラッグ移動できること。
